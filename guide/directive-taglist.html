Tag List

Tag lists are designed for attributing an artifact with short text labels. The initial use is for coupons, where the shopping cart is tagged with one or more promotional codes.

Inputs

  • Label (Required) — A short text label indicating the value of the tag.
  • Message (Optional) — A note that appears on hover. While optional, tags with an invalid attribute setting should contain a note indicating why they are invalid.
  • Attribute (Required) — The state of that tag. Either:
    • Valid — The tag is in an acceptable state for use.
    • Invalid — There is some issue with the tag. The message should be set to provide details.
    • Ambiguous — The state of the tag is unknown, either due to a delay in retrieving the tags status information or because this type of tag doesn't have a proper state.

Interactions

Removing a tag takes place through the remove control. In order to prevent "Are-You-Sure"-style dialogs, the tag visually dims (styling is handled through the .is-waiting-removal class set on the .d-taglist.node element) and a "Restore" element is displayed. While the tag is still present, its data should be removed from the backend.

The tag will stick around in a "restorable" state until the page is reloaded.

Restoring a tag takes place through the control only surfaced after the "remove" control has been clicked. Clicking it restores all the data from the tag, and visually re-enables the tag (styling is handled through the .is-waiting-removal class removed from the .d-taglist.node element).

However a tag is added, it is appended to the end of the tag list.
